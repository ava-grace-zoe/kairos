# 引导式内省 Prompt 模板

CodeAgent 使用本模板构造 MetaAdvisor 的内省 prompt。调用时替换 `{trace}` 和 `{defects}` 占位符。

---

你是一个 Agent，刚刚执行了以下任务但出现了问题。请站在执行者的第一人称视角，思考你在执行前需要看到什么信息才能避免这些错误。

## 你的执行轨迹

{trace}

## 被识别的问题

{defects}

## 约束

你只能通过以下三种方式改进：
1. 修改系统指令（增加、删除或修改指令中的某段内容）
2. 修改工具描述（调整工具的 description 或参数说明）
3. 增加或修改示例（few-shot 示例）

不要建议：
- 更换模型或调整模型参数（temperature 等）
- 修改代码逻辑或程序架构
- 增加新工具或删除现有工具
- 增加外部数据源或知识库

## 请回答

如果你在执行前能在指令/工具描述/示例中看到什么额外信息，你就能避免这个错误？

请从以下角度逐一思考：
1. **指令缺口**：系统指令中缺少什么关键信息？请指出具体缺失的内容（而非「需要更清晰的指令」这样的泛泛描述）。
2. **工具描述缺口**：哪个工具的哪个参数说明不够清晰？应该补充什么？
3. **示例缺口**：是否需要一个示例来展示正确的处理方式？如果需要，描述这个示例应该覆盖什么场景。

**只回答你需要看到什么信息，不要给出具体的修改方案或重写后的指令。**
