# progress-manager

本 skill 管理项目级工作进度。目标是把当前会话中已经探索清楚的代码认知、执行计划、下一步动作固化下来，让后续会话可以直接接手，少做重复搜索。

## 动作路由

本 skill 是统一入口，具体执行细则拆分为两个参考文档（references），按意图分流后读取对应细则执行：

- 用户要保存、记录、更新当前进度时：读取并执行**保存任务细则** [save-progress.md](save-progress.md)。
- 用户要恢复、继续、查看上次进度时：读取并执行**恢复任务细则** [resume-progress.md](resume-progress.md)。
- 用户意图不明确时：先根据语义判断是 save 还是 resume；仍不明确再简短询问。

## 共享约定

- 进度目录使用命令执行所在目录下的 `.work-progress/`。
- 未完成任务放在 `.work-progress/`。
- 完成任务放在 `.work-progress/archive/`。
- 不维护额外状态目录；暂停、卡住、风险写入进度文件内容。
- 文件名格式由 `save-progress.md` 定义；状态以目录位置为准，文件头部状态只是便于扫描的摘要。

## 匹配优先级

保存和恢复都按同一优先级定位目标文件：

1. **文件名优先**：用户给出完整文件名时，只操作该文件。
2. **关键词次优先**：用户给出任务关键词时，只在匹配该关键词的进度文件中选择。
3. **自动选择最后**：用户未给文件名或关键词时，按子流程规则自动新建、更新或恢复。

恢复时如果找不到明确目标，直接返回“无可恢复项”或列出候选让用户选择；不要猜测，也不要为了恢复而新建文件。

## 质量约束

- 保存时优先服务下次恢复，不写流水账。
- 恢复时优先复用进度文件中的代码认知资产，不全库重新探索。
- `下一步` 必须是动作级语句，包含要改的文件、要跑的命令或要验证的对象。
- 如果任务卡住，`当前卡点 / 风险` 必须写出解除条件；无卡点时写“暂无”。
- 默认不删除历史文件；只有用户明确要求清理时才删除。
- 文件内容细则以对应子文档为准，避免在 `SKILL.md` 里重复维护模板。
