# session-reflect 执行规范

按以下三阶段执行，不跳步。

目标：提取用户级稳定偏好，生成 AI 可直接消费的行为指令。不做项目级复盘。

## 一、初始化

1. 检查 `~/.agents/context.md` 是否存在且可读。
2. 若不存在，执行 `skills/session-reflect/scripts/setup.sh` 初始化。
3. 初始化后再次读取，若仍不存在则停止并报告失败原因。
4. 检查是否包含核心章节：`## 基本信息`、`## 综合画像`、`## AI 应对指南`、`## 偏好与习惯`。缺失则按以下结构补齐，不删除已有内容：
   ```markdown
   # User Context

   ## 基本信息

   - 名称：
   - 主要语言：
   - 常用技术栈：
   - 沟通语言：简体中文

   ## 综合画像

   暂无数据，完成首次会话反思后自动生成。

   ## AI 应对指南

   暂无数据。

   ## 偏好与习惯

   暂无数据。
   ```
5. 检查 `~/.agents/history.md` 是否存在，不存在则创建空文件。

## 二、会话分析

### 2.1 证据提取

1. 回顾当前会话，先提取用户级事实证据再给结论。
2. 仅提取跨项目可复用的证据，不记录项目私有细节。
3. 证据来源仅限当前会话中的用户交互行为（用户发送的消息、决策、反馈），不含项目配置文件（如 CLAUDE.md、AGENTS.md）中的预设内容。配置文件是项目级约束，不等于用户级稳定偏好。
4. 至少提取五类证据：
   - 沟通偏好：语言、简洁度、是否接受解释、是否要先执行后解释。
   - 表达模式：用户表达意图时的结构特征——习惯省略哪些上下文、哪些信息依赖 AI 从会话历史主动补全、是否存在反复出现的歧义模式。此类特征对 AI 意图理解对齐至关重要。
   - 协作偏好：是否需要确认、是否偏好直接落地、是否要求同步文档。
   - 工程偏好：脚本/工具偏好、测试偏好、格式与规范偏好。
   - 决策与风险偏好：速度优先或稳健优先、可接受的验证强度、容错边界。

### 2.2 生成 context.md 内容

基于证据直接生成以下章节内容：

5. `## 综合画像`：用户级稳定特征，每条有会话证据支撑，不做臆断。可包含编程风格、问题拆解特征、认知边界特征、提问反模式、表达模式。与已有画像合并，删除过时描述。
6. `## AI 应对指南`：将证据转化为可执行的 AI 行为指令。包含正向行为校准和硬性禁区（用户明确反感的 AI 行为）。
7. `## 偏好与习惯`：具体可执行偏好（如"文档使用简体中文""偏好 shell 脚本"）。合并重复项，保持精简。
8. 排除项（禁止写入）：项目私有目标、模块实现细节、一次性 workaround、单次任务指令。

## 三、写回与历史记录

1. 覆盖写回 `~/.agents/context.md`，章节顺序：基本信息 → 综合画像 → AI 应对指南 → 偏好与习惯。
2. 向 `~/.agents/history.md` **追加**一条记录（不读取全文件，直接 append），格式：
   ```markdown
   ### vYYYYMMDD-HHMMSS
   - 时间：YYYY-MM-DD HH:MM:SS
   - 会话摘要：（一句话）
   - 关键证据：（本次提取的核心证据，简要列出）
   - 画像/偏好变更：（变更了什么）
   ```
3. 写回校验：context.md 为有效 Markdown，无截断；history.md 存在本次新增条目。
4. 若写回失败，回退到写回前内容并报告失败。

## 输出要求

执行完成后，输出精简结果：

1. 初始化是否成功
2. 本次版本号
3. 更新了哪些画像与偏好
4. 是否存在待用户处理项
