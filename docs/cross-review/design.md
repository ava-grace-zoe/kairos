# cross-review 设计方案

## 设计目标

1. 最少工具调用次数完成完整评审流程（目标 1-2 次）
2. 执行模型主动构建自包含的评审上下文，不依赖用户手动指定
3. 关注点根据场景动态推导，不硬编码固定维度
4. 支持任意执行环境（不假设自身是哪个模型）

## 方案概览

```
用户触发 cross-review
        │
        ▼
  ┌─ 准备 + 分发 ──────────────────────────┐
  │  1. 收集最小充分评审上下文                │
  │  2. 根据场景推导关注点                    │
  │  3. 识别自身架构，排除同架构 agent         │
  │  4. 检测可用外部 agent CLI               │
  │  5. 构建 prompt，调用外部 agent          │
  │     └─ stdout 捕获结果                   │
  └──────────────────────────────────────────┘
        │
        ▼
  ┌─ 综合输出 ──────────────────────────────┐
  │  对比各 agent 结果 → 对话中直接输出       │
  │  共识 / 分歧 / 行动建议                  │
  └──────────────────────────────────────────┘
```

## 关键取舍

### 1. 评审上下文：最小充分原则

**旧方案**：用户指定文件就传文件，机械搬运。

**新方案**：执行模型主动构建完成评审所需的最小充分上下文——被评审对象本身、做出该决策的背景与约束、以及外部 agent 理解问题所必需的依赖信息。不足则主动从代码库/对话历史中补充，而非要求用户提供。

理由：外部 agent 零上下文，执行模型是信息桥梁。桥梁的职责不是转发用户的话，而是构建一个自包含的评审输入。

### 2. 关注点：场景感知推导

**旧方案**：固定五维度（正确性/可维护性/安全性/性能/遗漏）。

**新方案**：执行模型根据待评审内容的性质（新功能/修复/重构/安全/性能等），推导本次评审应重点关注的维度，作为用户指定关注点的补充。不硬编码场景-维度映射。

理由：模型本身有推导能力，硬编码映射表不可扩展且容易过时。描述意图比规定步骤更适合 skill 指令。

### 3. 信息流转：stdout 优先，文件降级

**旧方案**：文件系统作为 agent 间唯一信息通道（instructions.md → review-*.md → synthesis.md）。

**新方案**：

| 方向 | 默认方式 | 降级方式 |
|------|---------|---------|
| 上下文 → 外部 agent | prompt 内联 | 超长内容落 `$TMPDIR` 临时文件 |
| 外部 agent → 执行模型 | stdout 捕获 | 写文件（仅当 stdout 不可靠时） |
| 执行模型 → 用户 | 对话直接输出 | — |

理由：
- prompt 内联省掉 instructions.md 的写入+读取
- stdout 捕获省掉 review-*.md 的写入+读取
- synthesis.md 无需落盘，直接在对话中输出
- 调用次数从 ~7 次降到 1-2 次

### 4. 流程阶段：三阶段 → 两阶段

**旧方案**：prepare → dispatch → synthesize，三阶段各自产生中间文件。

**新方案**：准备+分发（合一） → 综合输出。

理由：prepare 和 dispatch 之间无需人工介入，合并后减少中间产物。synthesize 不再生成文件，直接输出。

## Agent 分发规则

### 可用 Agent 列表

| Agent | CLI 命令 | 模型架构 | 关键参数 |
|-------|----------|---------|---------|
| Claude | `claude -p "{prompt}"` | Claude 系列 | `--output-format text` |
| Codex | `codex exec "{prompt}"` | GPT 系列 | `--full-auto` |

### 分发逻辑

1. 识别自身所属模型架构
2. 排除同架构 agent
3. `command -v {cli}` 检测可用性（合并在调用 bash 中，不单独调用）
4. 调用可用外部 agent，超时 120s，失败则跳过并标注

### 退化场景

- 仅一个外部 agent 可用 → 跳过交叉对比，输出该 agent 结果 + 自身判断
- 无外部 agent 可用 → 提示用户安装，终止
- 无法确定自身身份 → 列出可用 CLI 的 agent，由用户选择

## 调用次数分析

| 场景 | 工具调用 | 说明 |
|------|---------|------|
| 待评审内容在文件系统 | 1 次 bash | prompt 内联引用路径 + 调用外部 agent + stdout 捕获 |
| 待评审内容在对话中 | 1 次 write + 1 次 bash | 上下文落临时文件 + 调用外部 agent + stdout 捕获 |

## 风险与缓解

| 风险 | 概率 | 缓解 |
|------|------|------|
| 外部 agent CLI 未安装 | 中 | `command -v` 检测，不可用则跳过并提示 |
| agent 无文件读写权限 | 中 | codex 需 `--full-auto`；仅在降级场景涉及文件 |
| stdout 输出被截断或混入日志 | 低 | 降级为文件回传 |
| prompt 超长（argv 限制） | 低 | 超长内容降级为临时文件引用 |
