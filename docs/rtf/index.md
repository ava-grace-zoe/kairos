# rtf（review-test-fix）

问题驱动的审查-测试-修复闭环技能：审查代码发现问题，用测试复现确认，自动修复并回归验证。

## 概述

- 这个 skill 是什么：一个将代码审查、测试复现、自动修复串联为闭环的执行流程
- 解决什么问题：避免「审查发现问题但不确定是否真实」和「修复引入新问题」的常见困境
- 适用场景：
  - PR / commit 的代码审查与修复
  - 对指定功能模块的健壮性检查
  - 用户说「帮我审查并修复」「rtf」「review-test-fix」时触发
- 不适用场景：
  - 纯架构评审（无具体代码可测试）
  - 性能基准测试（需要专门的 benchmark 工具）
  - 安全审计（需要专业安全扫描工具配合）

## 工作流程

1. **探查**：确定审查范围（代码变更 or 功能模块），探查项目测试基础设施
2. **审查**：多维度审查代码，产出结构化问题列表（severity + evidence + reproduction_hint）
3. **测试-修复循环**（逐问题执行）：
   - 测试设计：根据问题设计最小化复现测试
   - 测试执行：运行测试确认问题真实存在（未复现则标记误报）
   - 自动修复：最小化修复 + 修复验证（最多 3 次尝试）
   - 回归验证：运行已有测试确保不引入新问题
4. **报告**：输出综合报告（问题清单 + 变更 diff + 未解决项）

## 已知问题

1. 问题描述：测试设计质量依赖 agent 对问题的理解深度
   - 影响范围：可能导致 false positive 率偏高
   - 临时规避：`reproduction_hint` 字段引导测试设计方向
   - 后续计划：积累实际使用数据后优化审查-测试映射策略

2. 问题描述：无测试框架项目的验证脚本可靠性较低
   - 影响范围：shell 脚本难以覆盖复杂的边界条件
   - 临时规避：脚本执行错误时标记 `test_error` 而非误判
   - 后续计划：考虑为常见语言提供轻量级测试模板
