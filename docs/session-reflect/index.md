# session-reflect

用户级会话反思、持久化性格画像与高频工作流 Skill 化维护。

## 概述

在会话结束或用户明确要求反思时，自动完成以下四件事：

1. **初始化并校验** `~/.agents/context.md`。
2. **提取 A 轨（偏好与工作流）+ B 轨（性格与缺陷）双轨证据**，更新并持久化用户画像与性格缺陷。
3. **将变更历史与效能建言追加**到独立的 `~/.agents/history.md`。
4. **输出元认知反馈**（照镜子）与 **💡 工程效能建议**（如果发现高频、重复的流程，主动询问是否固化为 Skill 联动 `skill-creator`）。

`~/.agents/context.md` 通过软链接分发到多个编程助手工具，作为跨工具共享的用户级配置事实源。该 Skill 默认排除项目私有实现细节，仅沉淀跨项目稳定偏好、持久化性格与缺陷以及防错动作。

扩展文档：
- 设计文档：`design.md`
- 讨论记录：`discussion.md`
- 测试方案：`test-plan.md`
- 变更日志：`changelog.md`

## 工作流程

1. **初始化阶段**：读取 `~/.agents/context.md`，不存在则执行 `scripts/setup.sh`。校验核心章节（协作风格/性格与思维轮廓/行为校准/性格与认知防护/硬性禁区等）是否齐全，缺失则按 `SKILL.md` 的内联结构补齐。
2. **会话分析阶段**：
   - **A 轨提取 7 类偏好与工作流证据**：沟通偏好、表达模式、协作偏好、工程偏好、决策与风险偏好、问题解决工作流，以及**高频重复工作流（A7）**。
   - **B 轨提取 5 类性格与缺陷证据**：决策性格、认知缺陷与偏误（如注意力盲区、时区盲区等）、知识边界与防御姿态、反馈接受与情绪弹力、行为习惯惯性。
   - **双轨证据驱动生成/更新**：协作风格、性格与思维轮廓（**持久化写入缺陷**）、行为校准、性格与认知防护（**针对缺陷制定 AI 主动防御指令**）、硬性禁区及偏好与习惯。
3. **输出与写回阶段**：
   - 生成**元认知反馈**与**💡 工程效能建议**（直接输出给用户，不写入 `context.md`）。
   - 覆盖写回 `context.md`。
   - 追加写入 `history.md`（含 A/B 轨证据、变更内容和效能建言）。

## 已知问题

1. **软链接与已有配置冲突**：`setup.sh` 当前策略是跳过并警告，仍需用户手动合并。
2. **Cursor 暂无方案**：Cursor 全局规则存储在 SQLite，无法直接复用软链接分发策略。
3. **工作流自动落地局限性**：当前“工程效能建议”仅提示流程与设计思路，仍需用户调用 `/goal` 或 `skill-creator` 手动确认生成，无法完全 100% 静默无感生成并安装。
4. **版本回滚未脚本化**：尚未在 `scripts/` 中提供自动回滚工具。
